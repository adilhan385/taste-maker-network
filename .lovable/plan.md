

# План: Отзывы после покупки

## Что будет сделано

Покупатели смогут оставлять отзывы (рейтинг + комментарий) на каждый товар из доставленного заказа. Кнопка "Оставить отзыв" появится в карточке заказа со статусом "delivered".

## Изменения

### 1. Новый компонент `src/components/orders/ReviewDialog.tsx`
- Модальное окно с формой отзыва
- Выбор рейтинга (1-5 звёзд, кликабельные)
- Текстовое поле для комментария
- Отправка в таблицу `reviews` (уже существует в БД)
- Валидация: рейтинг обязателен, комментарий опционален
- После отправки кнопка меняется на "Отзыв оставлен"

### 2. Обновление `src/pages/Orders.tsx`
- Для заказов со статусом `delivered` — показать кнопку "Оставить отзыв" рядом с каждым товаром
- Загрузка существующих отзывов пользователя, чтобы не показывать кнопку повторно
- Интеграция компонента `ReviewDialog`

### 3. Обновление `src/lib/i18n.ts`
- Добавить ключи перевода:
  - `reviews.leaveReview` — "Оставить отзыв" / "Leave a review" / "Пікір қалдыру"
  - `reviews.rating` — "Оценка" / "Rating" / "Бағалау"
  - `reviews.comment` — "Комментарий" / "Comment" / "Пікір"
  - `reviews.submit` — "Отправить" / "Submit" / "Жіберу"
  - `reviews.submitted` — "Отзыв оставлен" / "Review submitted" / "Пікір қалдырылды"
  - `reviews.thankYou` — "Спасибо за отзыв!" / "Thank you for your review!" / "Пікіріңізге рахмет!"

### 4. Отображение реальных рейтингов в каталоге
- Обновить `src/pages/Catalog.tsx` — подсчитывать средний рейтинг и количество отзывов из таблицы `reviews` вместо захардкоженных значений

- `user_id`, `product_id`, `order_id`, `rating` (1-5), `comment`, `created_at`
- RLS: пользователи могут создавать/редактировать/удалять свои отзывы, все могут читать

Миграции БД не нужны — таблица уже готова.
