

# Аудит мобильной версии: найденные баги и недочёты

## Проблемы

### 1. DishCard — кнопки "Купить" и "Написать" видны только при hover
**Файл:** `src/components/catalog/DishCard.tsx` (строка 150-153)
- `opacity-0 group-hover:opacity-100` — на тач-экранах hover не работает
- Покупатель на телефоне не видит кнопку "Добавить в корзину" и "Написать повару"

**Исправление:** Убрать `opacity-0 group-hover:opacity-100`, показывать кнопки всегда. Или добавить `md:opacity-0 md:group-hover:opacity-100` чтобы hover работал только на десктопе.

### 2. Панель повара — табы не скроллятся на мобильных
**Файл:** `src/pages/ChefDashboard.tsx` (строка 103-133)
- `TabsList` с `flex-wrap` — на узком экране табы переносятся на несколько строк и занимают слишком много места
- Нет `overflow-x-auto` для горизонтального скролла

**Исправление:** Заменить `flex-wrap` на `overflow-x-auto flex-nowrap` с `min-w-max`.

### 3. Navbar — на мобильном слишком много элементов в правой части
**Файл:** `src/components/layout/Navbar.tsx` (строки 145-247)
- LanguageSelector + WalletWidget + Cart + Bell + UserMenu + Burger = 6 элементов в одной строке
- На экране 320-375px всё сжимается и может выходить за пределы

**Исправление:** Спрятать WalletWidget и LanguageSelector на мобильных (`hidden sm:flex`), перенести их в мобильное меню.

### 4. Cart — на мобильном карточки товаров обрезаются
**Файл:** `src/pages/Cart.tsx` (строка 266-285)
- `flex gap-4` с фиксированным `w-24 h-24` изображением — на узких экранах текст может не помещаться
- Кнопки +/- и цена могут наслаиваться

**Исправление:** На мобильных сделать `flex-col` layout для карточек или уменьшить изображение до `w-16 h-16`.

### 5. Orders — на мобильном кнопки действий сжимаются
**Файл:** `src/pages/Orders.tsx` (строка 291-298)
- Кнопка "Написать повару" с текстом и иконкой может не помещаться рядом с аватаром повара

**Исправление:** На мобильных показывать только иконку без текста (`hidden sm:inline`).

### 6. Wallet — таблица транзакций не адаптивна
**Файл:** `src/pages/Wallet.tsx` (строки 157-186)
- `<Table>` с 4 колонками на маленьком экране горизонтально переполняется
- Нет горизонтального скролла

**Исправление:** Обернуть таблицу в `overflow-x-auto`, или на мобильных заменить на карточный вид.

### 7. Мобильное меню не закрывается при навигации
**Файл:** `src/components/layout/Navbar.tsx` (строки 250-269)
- При клике на ссылку в мобильном меню, меню остаётся открытым
- Нужно добавить `onClick={() => setMobileMenuOpen(false)}` на ссылки

## План исправления

| Файл | Что |
|------|-----|
| `src/components/catalog/DishCard.tsx` | Кнопки всегда видны на мобильных |
| `src/pages/ChefDashboard.tsx` | Горизонтальный скролл табов |
| `src/components/layout/Navbar.tsx` | Спрятать лишнее на мобильных, закрывать меню при навигации |
| `src/pages/Cart.tsx` | Адаптивные карточки товаров |
| `src/pages/Orders.tsx` | Компактные кнопки на мобильных |
| `src/pages/Wallet.tsx` | Горизонтальный скролл таблицы |
